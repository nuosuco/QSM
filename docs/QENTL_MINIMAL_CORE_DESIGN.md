# QEntL最小化核心设计
## 实现完全自主的量子自举

### 概述
为实现QEntL编译器的自举（用QEntL编译QEntL编译器），需要实现一个最小化的QEntL运行时核心。这个核心将提供最基本的QEntL语言功能，足以运行现有的编译器代码（2000+行QEntL），然后让编译器编译自身，实现完全自主的工具链。

### 设计目标
1. **完全自主**：不依赖任何第三方语言、库或运行时
2. **最小化**：只实现运行编译器所需的最基本功能
3. **可自举**：能运行编译器代码，编译器能编译自身
4. **可扩展**：为后续完善提供基础架构

### 核心组件
#### 1. 基础运行时（C语言实现，约500行）
```
minimal_qentl_core.c
├── 内存管理
│   ├── 简单内存分配器
│   ├── 垃圾回收标记
│   └── 字符串池管理
├── 类型系统
│   ├── 基础类型：整数、浮点数、布尔、字符串
│   ├── 数组类型支持
│   └── 对象类型支持
├── 执行引擎
│   ├── 字节码解释器
│   ├── 函数调用栈
│   └── 异常处理框架
└── 标准库
    ├── 基本I/O（控制台输入输出）
    ├── 数学函数
    └── 系统调用封装
```

#### 2. QEntL语言子集支持
**必须支持的语言特性**：
- ✅ 变量声明：`let x = 值`
- ✅ 基础类型：整数、浮点数、布尔、字符串
- ✅ 控制流：`如果`、`否则`、`循环`
- ✅ 函数声明：`函数 名称(参数) -> 返回类型 { ... }`
- ✅ 数组操作：`[1, 2, 3]`、`数组[索引]`
- ✅ 对象操作：`对象.属性`、`对象.方法()`
- ✅ 导入语句：`import "模块路径"`

**可选支持（编译器暂时不需要）**：
- ⏳ 量子操作符：`|>`、`@>`等
- ⏳ 异步编程：`async`/`await`
- ⏳ 高级类型系统：泛型、接口继承

#### 3. 字节码格式
**简化版QEntL字节码**：
```
; 示例字节码
.version 1.0
.quantum_gene QGC-MINIMAL-20260202

; 数据段
.data
  string_const_1 "Hello, QEntL!"
  int_const_1 42

; 代码段
.code
  main:
    LOAD_CONST string_const_1
    PRINT
    LOAD_CONST int_const_1
    RETURN
```

**指令集设计**：
```
基础指令：
  LOAD_CONST <常量索引>    ; 加载常量
  LOAD_VAR <变量索引>      ; 加载变量
  STORE_VAR <变量索引>     ; 存储变量
  CALL_FUNCTION <函数索引> ; 调用函数
  RETURN                  ; 返回值

控制流指令：
  JUMP <目标地址>          ; 无条件跳转
  JUMP_IF_TRUE <地址>     ; 条件为真跳转
  JUMP_IF_FALSE <地址>    ; 条件为假跳转

运算指令：
  ADD, SUB, MUL, DIV      ; 算术运算
  EQ, NEQ, GT, LT         ; 比较运算
  AND, OR, NOT            ; 逻辑运算
```

### 技术挑战与解决方案
#### 挑战1：自举循环依赖
**问题**：需要QEntL运行时运行QEntL编译器，但编译器编译运行时
**解决方案**：手写最小化核心（C语言）→ 运行编译器 → 编译器编译自身 → 替换手写核心

#### 挑战2：语言特性完备性
**问题**：编译器使用了哪些QEntL特性？
**解决方案**：分析现有编译器代码（`quantum_compiler_v2.qentl`），提取必需的语言特性子集

#### 挑战3：性能要求
**问题**：编译器代码可能复杂，需要足够性能
**解决方案**：优化关键路径，使用高效数据结构，后续可进行性能优化

### 与现有编译器的集成
#### 1. 编译流程调整
```
现有：QEntL源码 → QEntL编译器（无法运行） → 字节码
调整：QEntL源码 → 最小化核心运行编译器 → 字节码
目标：QEntL源码 → QEntL编译器（自编译版本） → 字节码
```

#### 2. 接口设计
```c
// 最小化核心API
void qentl_init();                          // 初始化运行时
void qentl_load_bytecode(const char* code); // 加载字节码
void qentl_execute();                       // 执行字节码
void qentl_cleanup();                       // 清理资源

// 编译器集成接口
char* qentl_compile_source(const char* source_code);
```
