# SOM 松麦 - 项目总方案

> 有机认证 · 药食同源 · 健康环保 · 中医养生

## 项目概述

**项目名称：** SOM（松麦）  
**域名：** som.top  
**助手名称：** 小麦SOM  
**定位：** 中医辨证 + 有机认证药食同源 + 游戏化养生的健康生活平台

**核心理念：**
- 不卖货，卖"对症的养生方案"
- 不教课，让用户在玩中学会养生
- 不强调技术，用户感受到的是一个懂你的养生伙伴

**产品标准：**
- 只做有机认证的有机食品
- 只做药食同源食材（有正规资质）
- 只做健康环保认证的健康产品
- 不做无资质证明的"原生态"产品

## 技术架构

### 核心引擎

```
小麦SOM = Qwen3.5 2B + RAG知识库
```

- **Qwen3.5 2B**：本地部署，负责自然语言理解与生成
- **RAG知识库**：中医辨证逻辑 + 药食同源食材库 + 体质辨识体系 + 用户对话数据
- 所有功能模块共享同一引擎和知识库

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端-微信小程序 | 原生小程序框架 | 微信生态，用户基数大 |
| 前端-网页版 | Vue 3 / React | som.top，与小程序功能同步 |
| 后端服务 | Node.js / Python FastAPI | 统一API，同时服务小程序和网页 |
| 模型推理 | Qwen3.5 2B + llama.cpp | 本地部署，数据自主 |
| 知识库 | RAG（向量数据库 + 知识图谱） | 中医辨证 + 药食同源 |
| 数据库 | SQLite / PostgreSQL | 用户数据、对话记录、积分系统 |
| 商品对接 | 淘宝联盟API + 京东联盟API + 拼多多联盟API | 三平台佣金 |

### 部署方案

**第一阶段（当前服务器）：**
- 服务器：4核CPU / 8GB内存 / 无GPU
- Qwen3.5 2B 量化部署（Q4_K_M，约1.5GB）
- 足够支撑前期用户

**第二阶段（用户量增长后）：**
- 升级服务器或租用GPU实例
- 部署更大模型（7B级别中医专用模型）
- 知识库持续扩充

│   └── 每日中医小考题 + 节气食疗方案

```
用户进入SOM（小程序或网页）
    ↓
与小麦SOM对话，描述身体状况
    ↓
小麦SOM辨证分析，判断体质与证型
    ↓
生成个性化养生方案（食疗+生活建议）
    ↓
方案中包含药食同源产品推荐（淘宝客/京东联盟链接）
    ↓
用户可进入养生谷、护眼训练等游戏模块
    ↓
游戏中学习中医知识，赚取积分
    ↓
积分兑换产品优惠券，促进复购
    ↓
小麦SOM定期跟踪，调整方案
```

| 来源 | 说明 | 阶段 |
|------|------|------|
| 淘宝客佣金 | 用户通过推荐链接在淘宝购买，SOM获得佣金（食品类5%-15%） | 第一期 |

| 增值服务 | 高级养生方案、私人定制等 | 远期 |

| 项目 | 成本 | 说明 |
|------|------|------|

## 数据自主与隐私

### 核心原则

- 所有用户健康数据存储在自有服务器
- 模型本地部署，数据不经过第三方
- 用户对话数据、辨证记录、体质画像完全归SOM所有

### 数据壁垒

```
用户对话 → 辨证记录 → 体质画像 → 购买偏好 → 反馈数据
    ↓
持续积累 → 知识库越来越丰富 → 辨证越来越准 → 用户越来越信任
    ↓
这就是SOM的核心壁垒，别人抄不走
```

## 分期计划

### 第一期：MVP（最小可用产品）

**目标：** 验证核心模式，跑通"辨证→推荐→成交"链路

**功能：**
1. **小麦SOM问诊** — 对话式中医辨证，生成养生方案
2. **每日养生签到** — 每天一道养生小考题，积分奖励
3. **产品推荐** — 对接淘宝客/京东联盟，方案中嵌入商品链接

**交付物：**
- 微信小程序（基础版）
- som.top 网页版（基础版）
- 后端API服务
- RAG知识库（药食同源 + 基础辨证）

---

### 第二期：游戏化

**目标：** 提升用户粘性，日活增长

**功能：**
1. **护眼训练** — 眼球运动小游戏 + 明目食疗推荐
2. **节气养生挑战** — 24节气知识问答 + 节气食疗方案
3. **积分商城** — 积分兑换优惠券/产品

---

### 第三期：养生谷

**目标：** 打造核心游戏IP，形成差异化壁垒

**功能：**
1. **养生谷** — 种药材、辨证问诊、角色成长
2. **社区系统** — 用户分享养生经验、排行榜
3. **师徒系统** — 老用户带新用户

---

### 第四期：自有品牌

**目标：** 利润最大化

**功能：**
1. 核心品类自有品牌（红枣、枸杞、茯苓、陈皮等）
2. 代工厂贴牌
3. 供应链直供

## RAG知识库核心内容

### 药食同源食材库

基于国家卫健委公布的药食同源目录，包含：
- 食材名称、性味归经
- 功效与适应症
- 禁忌与注意事项
- 常见搭配（配伍）
- 推荐食谱

### 体质辨识体系

基于中医九种体质分类：
- 平和质、气虚质、阳虚质、阴虚质
- 痰湿质、湿热质、血瘀质、气郁质、特禀质
- 每种体质的特征、易患疾病、调养原则

### 辨证逻辑规则

- 症状 → 证型 的映射规则
- 多症状组合的辨证逻辑
- 证型 → 食疗方案 的推荐规则
- 特殊人群（孕妇、儿童、老人）的禁忌规则

## 风险与应对

| 风险 | 应对 |
|------|------|
| 辨证建议不准确 | RAG知识库严格基于权威资料，加免责声明 |
| 用户健康数据泄露 | 本地部署，数据不出服务器 |
| 淘宝客/京东联盟政策变化 | 双平台对冲风险，后期转自有品牌 |
| 服务器性能不足 | 前期用户少够用，后期按需升级 |
| 合规风险（医疗建议） | 定位为"养生建议"而非"医疗诊断"，加声明 |
