# QEntL软件分发机制设计

## 🎯 Web版本的三重角色

### 1. 在线服务
用户直接通过浏览器使用QEntL量子操作系统，无需安装。

### 2. 下载中心
用户从这里下载安装包：
- 量子虚拟机安装包（Windows/Mac/Linux）
- 原生操作系统镜像（各种设备）
- 更新包

### 3. 自动更新服务
已安装用户自动接收更新推送。

## 🔄 版本发布流程

### 新版本开发完成
```bash
# 1. 开发完成，版本号升级
版本: v1.0.0 → v1.0.1

# 2. 打包
cd /root/QSM/VM
npm run build          # 构建虚拟机
npm run package        # 打包安装程序

# 3. 复制到Web下载目录
cp dist/QEntL-VM-Setup-v1.0.1.exe /root/QSM/Web/downloads/vm/windows/
cp dist/QEntL-VM-v1.0.1.dmg /root/QSM/Web/downloads/vm/macos/
cp dist/QEntL-VM-v1.0.1.AppImage /root/QSM/Web/downloads/vm/linux/

# 4. 更新版本清单
echo '{"version":"1.0.1","releaseDate":"2026-03-07"}' > /root/QSM/Web/update/manifest.json

# 5. 升级Web服务
bash /root/QSM/scripts/upgrade_web.sh
```

### 用户更新流程

#### 在线用户
```
Web服务升级 → 用户刷新页面 → 直接使用新版本
```

#### 新用户下载
```
访问 som.top/downloads → 选择版本 → 下载安装包 → 安装使用
```

#### 已安装用户
```
启动软件 → 检查更新 → 发现新版本 → 自动下载 → 安装更新
```

## 📡 自动更新API

### 版本检查 (check.js)
```javascript
// 用户软件启动时检查
const currentVersion = "1.0.0";
const response = await fetch("https://som.top/update/manifest.json");
const { version, releaseDate, changelog } = await response.json();

if (version !== currentVersion) {
    // 提示用户更新
    showUpdateNotification(version, changelog);
}
```
