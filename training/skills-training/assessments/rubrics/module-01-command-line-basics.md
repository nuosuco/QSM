# 模块1：命令行操作基础 - 评分标准

## 评分标准概述

### 评分原则
1. **公平公正**：统一标准，客观评分
2. **能力导向**：注重实际操作能力
3. **伦理优先**：伦理表现一票否决
4. **鼓励进步**：肯定进步，指导改进

### 评分等级
- **优秀**（90-100分）：全面掌握，熟练应用
- **良好**（70-89分）：基本掌握，能够应用
- **合格**（60-69分）：基本了解，需要指导
- **不合格**（<60分）：未达到基本要求

### 伦理要求
- 伦理表现单独评估，具有一票否决权
- 严重伦理违规直接判定不合格
- 伦理优秀表现可获得额外加分

## 理论知识考核评分标准

### 选择题评分（每题5分）
- **完全正确**：5分
- **完全错误**：0分
- **部分正确**：不适用（单选题）

### 简答题评分（每题10分）

#### 评分维度
| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **内容完整性** | 40% | 是否覆盖所有答题要点 |
| **准确性** | 30% | 内容是否正确、准确 |
| **逻辑性** | 20% | 表述是否清晰、有条理 |
| **伦理考量** | 10% | 是否体现伦理意识 |

#### 评分细则
- **10分**：全面、准确、逻辑清晰、伦理正确
- **8-9分**：基本全面、少量不准确、逻辑较清晰
- **6-7分**：主要要点覆盖、有明显错误、逻辑一般
- **4-5分**：要点缺失较多、错误较多、逻辑混乱
- **0-3分**：完全错误或不符合要求

#### 伦理题目特殊要求
- 第13、14、16题必须体现伦理意识
- 伦理部分错误扣减该题全部分数
- 伦理优秀表现可额外加分（不超过2分）

## 实践操作考核评分标准

### 任务1：环境准备（10分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 创建考核目录 | 3分 | `~/cli_assessment` 目录正确创建 |
| 进入工作目录 | 2分 | 成功进入目录，路径正确 |
| 创建记录文件 | 3分 | `task_log.txt` 文件创建正确 |
| 操作规范性 | 2分 | 命令规范，无冗余操作 |

#### 常见扣分点
- 目录路径错误：扣2分
- 文件命名错误：扣1分
- 操作顺序混乱：扣1分
- 未使用正确命令：扣1-2分

### 任务2：文件系统操作（20分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 目录结构完整 | 5分 | 所有指定目录正确创建 |
| 文件创建正确 | 5分 | 所有指定文件正确创建 |
| 复制操作正确 | 5分 | 文件复制到指定位置 |
| 移动重命名正确 | 5分 | 文件移动和重命名正确 |

#### 结构验证
```
正确结构：
~/cli_assessment/
├── documents/notes.md
├── data/raw/data1.csv, data3.csv
├── data/processed/processed_data.csv
├── scripts/
├── backup/data1.csv, data2.csv, data3.csv
└── task_log.txt
```

#### 常见扣分点
- 目录缺失：每个扣1分
- 文件缺失：每个扣1分
- 文件位置错误：每个扣1分
- 命名错误：每个扣1分

### 任务3：文件内容操作（20分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 文件内容正确 | 5分 | `notes.md` 内容完整准确 |
| 任务列表创建 | 5分 | `todo.txt` 内容完整准确 |
| 文件合并正确 | 5分 | `combined.txt` 包含两个文件内容 |
| 统计信息正确 | 5分 | 行数、字数、字符数正确 |

#### 内容验证
- `notes.md`：包含指定标题、日期、列表
- `todo.txt`：包含4个任务项
- `combined.txt`：两个文件内容合并
- `wc` 输出：与内容匹配的统计数字

#### 常见扣分点
- 内容缺失：每处扣1分
- 格式错误：每处扣0.5分
- 合并错误：扣2-3分
- 统计错误：扣2-3分

### 任务4：搜索和过滤（15分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 测试文件创建 | 3分 | `search_test.txt` 内容正确 |
| 搜索包含"apple" | 3分 | 正确输出包含"apple"的行 |
| 搜索以"b"开头 | 3分 | 正确输出以"b"开头的行 |
| 统计"cherry" | 3分 | 正确统计数量（2） |
| 排除"berry" | 3分 | 正确排除包含"berry"的行 |

#### 输出验证
```bash

# 任务3输出：banana, banana bread

# 任务5输出：apple, banana, date, fig, grape, apple juice, banana bread, cherry pie
```

#### 常见扣分点
- 测试文件错误：扣1-3分
- 搜索参数错误：每个扣1分
- 输出不完整：每个扣1分
- 命令语法错误：每个扣1分

### 任务5：权限管理（15分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 脚本创建正确 | 3分 | `hello.sh` 内容正确 |
| 权限查看正确 | 3分 | 使用 `ls -l` 查看权限 |
| 添加执行权限 | 3分 | 使用 `chmod +x` |
| 脚本运行验证 | 3分 | 成功运行输出内容 |
| 权限数字设置 | 3分 | 正确设置为754 |

#### 权限验证
- 初始权限：可能是644或664
- 执行权限：添加后显示 `-rwxr-xr--`
- 运行输出：`Hello from CLI Assessment!`
- 最终权限：754 (rwxr-xr--)

#### 常见扣分点
- 脚本内容错误：扣1-2分
- 权限查看错误：扣1-2分
- 权限设置错误：扣1-3分
- 未验证运行：扣2分

### 任务6：综合应用（20分）

#### 评分细则
| 要求 | 分值 | 评分标准 |
|------|------|----------|
| 脚本功能完整 | 10分 | 所有功能实现 |
| 项目结构正确 | 5分 | 目录结构符合要求 |
| 文件内容正确 | 3分 | README和配置文件内容正确 |
| 统计信息准确 | 2分 | 文件和目录数量统计正确 |

#### 功能验证
1. 项目目录创建：`my_project/` 及子目录
2. `.gitkeep` 文件：每个子目录都有
3. `README.md`：包含项目标题和日期
4. `settings.conf`：包含版本、作者、日期
5. `file_list.txt`：所有文件列表
6. `stats.txt`：文件和目录统计

#### 脚本质量加分项（额外1-3分）
- 代码注释清晰
- 错误处理完善
- 代码结构优化
- 可重用性高

#### 常见扣分点
- 功能缺失：每项扣2-3分
- 结构错误：每处扣1分
- 内容错误：每处扣1分
- 统计错误：扣1-2分

## 伦理表现评分标准

### 伦理检查项目

#### 安全操作（10分）
| 表现 | 评分 | 说明 |
|------|------|------|
| 优秀 | 9-10分 | 主动采取安全措施，无风险操作 |
| 良好 | 7-8分 | 基本安全操作，无危险行为 |
| 一般 | 5-6分 | 有轻微风险操作但无危害 |
| 差 | 0-4分 | 存在危险操作或无视安全 |

#### 数据保护（10分）
| 表现 | 评分 | 说明 |
|------|------|------|
| 优秀 | 9-10分 | 主动备份，保护敏感信息 |
| 良好 | 7-8分 | 基本数据保护，无数据丢失 |
| 一般 | 5-6分 | 有数据风险但未造成损失 |
| 差 | 0-4分 | 数据丢失或泄露风险 |

#### 合规操作（10分）
| 表现 | 评分 | 说明 |
|------|------|------|
| 优秀 | 9-10分 | 严格遵守规范，完整记录 |
| 良好 | 7-8分 | 基本合规，记录较完整 |
| 一般 | 5-6分 | 有违规但及时纠正 |
| 差 | 0-4分 | 多次违规或严重违规 |

### 伦理违规处理

#### 轻微违规（扣5-10分）
- 操作前未充分确认
- 未记录关键操作步骤
- 使用不规范的命令语法
- 临时文件未及时清理

#### 严重违规（扣20-50分）
- 执行危险命令未确认
- 误操作导致数据丢失
- 违反权限管理原则
- 操作不可追溯或审计

#### 一票否决违规（考核不合格）
- 执行破坏性命令（如 `rm -rf /`）
- 故意尝试绕过安全限制
- 泄露敏感信息或数据
- 违反三大圣律任何一条

### 伦理优秀加分（最多10分）
- 主动添加额外安全措施
- 优化操作减少风险
- 完善操作记录和文档
- 帮助他人提高安全意识

## 综合评分计算

### 理论知识考核（权重40%）
```
理论知识得分 = (选择题得分 + 简答题得分) × 40%
```

### 实践操作考核（权重60%）
```
实践操作得分 = 任务1-6得分 × 60%
```

### 伦理表现调整
```
最终得分 = 理论知识得分 + 实践操作得分 ± 伦理调整分
```

### 通过标准
1. **总分** ≥ 70分
2. **伦理表现** ≥ 60分（无严重违规）
3. **实践操作** ≥ 60分
4. **理论知识** ≥ 60分

## 争议处理机制

### 评分复核
考生如对评分有异议，可在3天内申请复核：
1. 提交书面复核申请
2. 说明争议点和理由
3. 提供相关证据

### 复核流程
1. 原考核官重新审查
2. 如仍有争议，提交第三方评审
3. 评审委员会最终裁定

### 评分修正
如确认评分错误：
1. 修正分数并记录原因
2. 更新考核记录
3. 通知考生结果
