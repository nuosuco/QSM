# QSM量子叠加态架构设计文档 V3

## 核心理念：量子自举环境

**QSM项目的根本原则：不依赖任何第三方语言、代码、文件、环境。**

### 2. 自举三阶段

#### 阶段一：C启动器 + QVM内核

**C启动器** 是整个系统的唯一外部依赖，功能极简：

```c
// qvm_boot.c - 量子虚拟机启动器
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// 最小功能：加载QBC字节码文件，交给QVM执行
int main(int argc, char* argv[]) {
    if (argc < 2) {
        printf("用法: qvm_boot <kernel.qbc>\n");
        return 1;
    }
    
    // 1. 加载内核QBC文件
    QBCProgram* kernel = qbc_load(argv[1]);
    
    // 2. 初始化量子虚拟机
    QVMState* qvm = qvm_init(kernel);
    
    // 3. 执行内核 → 控制权交给QEntL环境
    qvm_run(qvm);
    
    // 4. 清理（正常情况下不会到这里）
    qvm_free(qvm);
    return 0;
}
```

**C启动器职责**：
- 读取内核QBC字节码文件
- 分配内存给QVM
- 调用QVM入口点
- **不包含任何业务逻辑**

**QVM内核**（用QEntL编写，编译为QBC）：
- 文件系统驱动
- 内存管理
- 进程调度
- 设备抽象

#### 阶段二：量子动态文件系统

QVM启动内核后，内核建立量子动态文件系统：

```
/qentl/              ← 量子文件系统根
  /system/           ← 系统目录
    /kernel/         ← 内核模块
    /compiler/       ← QEntL编译器（QEntL编写）
    /runtime/        ← 运行时库
  /models/           ← 量子模型
    /qsm/            ← QSM量子叠加态模型
    /som/            ← SOM量子经济模型
    /weq/            ← WeQ量子通讯模型
  /data/             ← 量子数据
  /users/            ← 用户空间
  /quantum/          ← 量子态空间
    /register/       ← 量子寄存器
    /entangle/       ← 纠缠态存储
```

#### 阶段三：完全自举

一旦量子动态文件系统和编译器就位：
- **编译器用QEntL重写**（不再依赖Python）
- **模型训练在QVM内执行**（不再依赖PyTorch）
- **API服务在QVM内运行**（不再依赖Flask）
- **一切在量子虚拟机构建的QEntL环境中运行**

### 3. 文件格式规范

| 格式 | 用途 | 说明 |
|------|------|------|
| `.qentl` | QEntL源码 | 量子编程语言源文件 |
| `.qbc` | QBC字节码 | 编译后的量子字节码 |
| `.qim` | 量子镜像 | 量子虚拟机系统镜像 |
| `.json` | 权重数据 | 仅用于模型权重序列化 |
| `.c` | C启动器 | 唯一的外部语言文件 |

**❌ 禁止使用的文件格式**：
- `.py` — Python脚本（开发过渡期可用，最终消除）
- `.js` — JavaScript
- `.sh` — Shell脚本（除系统引导外）
- `.pth` — PyTorch模型（迁移为.json权重）

### 6. QEntL量子编程语言规范

#### 6.1 基本语法

```
// 注释
配置 { 键: 值 }                    // 配置声明

类型 名称 { 字段: 类型 }           // 类型定义

函数 名称(参数: 类型) -> 返回类型 { // 函数定义
    如果 条件 { ... } 否则 { ... } // 条件
    循环 变量 在 范围 { ... }      // 循环
    返回 值                        // 返回
}

quantum_program 名称 {             // 量子程序
    setup: 函数() { ... }          // 初始化
    run: 函数() { ... }            // 主循环
}
```

### 7. 设计哲学

1. **自举原则**: 系统必须能用自身构建自身
2. **最小外部依赖**: 仅C启动器一个二进制文件
3. **量子优先**: 量子操作是一等公民，不是附加层
4. **渐进迁移**: 当前允许Python过渡，但必须规划消除路径
5. **文件格式纯化**: 最终只存在 .qentl / .qbc / .qim / .json / .c 五种格式
